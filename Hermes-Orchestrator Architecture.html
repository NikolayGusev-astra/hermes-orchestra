Hermes-Orchestrator Architecture

Оркестрация мульти-агентных проектов: один чистый Hermes управляет всеми

Оркестратор

  • • Чистый Hermes profile, одна роль: orchestrator
  • • Принимает задачи из CLI, Telegram, Discord, API
  • • Разбивает проекты на подзадачи (task_breakdown)
  • • Назначает исполнителей (task_assign)
  • • Собирает результаты, обновляет статусы
  • • Сохраняет всё в persistent SQLite store

Project Store (SQLite)

  • • Живёт вечно — не в памяти сессии
  • • Таблица projects: id, name, client, status
  • • Таблица tasks: id, project_id, parent_id, title
  • • assignee, priority, deadline, status
  • • История изменений для аудита
  • • Общая БД с сессиями (state.db)

Agent Fleet

  • • Hermes Workers: через delegate_task, N параллельно
  • • Claude Code / Codex: через ACP протокол
  • • OpenClaw / Gemini: через terminal
  • • Любой MCP-сервер: через MCP
  • • Результаты собираются обратно в orchestrator
  • • Статусы задач обновляются автоматически
